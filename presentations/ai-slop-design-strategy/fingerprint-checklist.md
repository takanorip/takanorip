# AIスロップ哲学チェックリスト

**用途**: AI生成またはAI支援で作ったUI/コピー/プロトタイプのレビュー  
**所要時間**: 約10分  
**目的**: 「AIっぽい見た目」を潰すことではなく、哲学なき生成になっていないかを確認する

---

## 1. Philosophy — 哲学はあるか

- [ ] このデザインが何を良い体験とみなしているか説明できる
- [ ] ユーザーのどんな不安を減らすのか言語化されている
- [ ] 何を約束しないのか、何を煽らないのかが決まっている
- [ ] 「これは違う」と判断する境界線がある
- [ ] 見た目ではなく、判断基準として説明できる

## 2. Brief — AIに渡す前の入力は十分か

- [ ] Goal / Audience / Context が書かれている
- [ ] 守るべき Principle が明記されている
- [ ] Avoid（やらない表現・避けるトーン）がある
- [ ] Success Criteria（何を満たしたら良いか）がある
- [ ] AIに任せてはいけない判断が明示されている

## 3. Boundary — AIに任せる領域は正しいか

- [ ] AIが方向性や価値判断を勝手に決めていない
- [ ] AIの出力を採用した理由を人間が説明できる
- [ ] AIは探索・展開・検証補助として使われている
- [ ] 最終的なトーン・優先順位・責任は人間が持っている
- [ ] 「AIがそう出したから」が採用理由になっていない

## 4. Output — 出力は哲学を体現しているか

- [ ] 情報の優先順位がユーザーの状況に合っている
- [ ] コピーが期待値を過剰に上げていない
- [ ] 不安を煽る表現になっていない
- [ ] 装飾が意味を持っている
- [ ] 自分たちらしい違和感のなさがある

## 5. Harness — ハーネスを過信していないか

- [ ] Lintやチェックリスト通過だけでOKにしていない
- [ ] DESIGN.md / rules / prompt が判断の代替になっていない
- [ ] 例外判断を人間が行っている
- [ ] ハーネスの内容が哲学・原則と接続している
- [ ] 「ルールは通っているが意思がない」状態を確認している

## 6. Classic Fingerprints — 表面症状も確認する

- [ ] Inter / Roboto / system-ui が安易な主フォントになっていない
- [ ] 紫〜インディゴのグラデーションが意味なく主役になっていない
- [ ] ヒーロー → 3等分カード → CTA のみの構成になっていない
- [ ] ガラスカード + グロー影が無目的に並んでいない
- [ ] 「革新的」「次世代」「シームレス」などの抽象語で埋まっていない

---

## 判定基準

| 状態 | 判定 | アクション |
|-----|------|-----------|
| 哲学・境界・評価基準を説明できる | Pass | 実装・検証へ進む |
| 見た目は整っているが判断基準が弱い | Warning | Brief / Principle を補強する |
| AIの初期出力を説明できず採用している | Slop | 方向性選定からやり直す |

---

## 修正の優先順位

1. **哲学を1文で書く** — 何を良い体験と考えるか
2. **AIに任せない判断を決める** — Why / トーン / 優先順位
3. **AI用ブリーフを作る** — Goal / Principle / Avoid / Success Criteria
4. **複数案を比較する** — 1発生成を採用しない
5. **採用理由を残す** — チームの学習資産にする
